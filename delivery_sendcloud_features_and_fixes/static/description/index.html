New features and improvements for delivery sendcloud API

This module allows you to extend the "delivery_sendcloud_official" and fixes some bugs

Configure KIT Product

    On occasions there are products that are configured as KITs, but at the time of shipping they may be described as a single product that includes all the subcomponents that they form. Using this module it will be possible to send the KIT product including the subproducts that make it up.
  • Install Delivery SendCloud KIT Product Module
  • Go to Sendcloud --> Configuration --> Integration and select your active Sendcloud Integration
  • You will find a "KIT product" checkbox. If it is checked the new function will be used.

Fixed Bugs

  • The price is modified, since in the case that the product is a KIT and the functionality of sending it as a single component is not activated, the price is not correctly taken. What has been done is to indicate the proportional part within the KIT based on the price that this component has.
  • The weight is modified, as with the price, it is not taken correctly if the product is part of a KIT
  • Modify the shipping methods so that it takes into account only those that meet the weight of the order

Important Things

    Remember to:
  • Put the HS Code in KIT product
  • Inform Weight field in KIT product. A very useful module is https://github.com/OCA/product-attribute/tree/14.0/product_weight you can find in v16 here https://github.com/vandekul/addons-zynthian16/tree/16.0/product_weight
